Adding RBAC to Tiller

In Kubernetes, granting a role to an application-specific service account is a best practice to ensure that your application is operating in the scope that you have specified.
You can add a service account to Tiller using the --service-account flag while you’re configuring Helm. As a prerequisite, you’ll have to create a role binding which specifies a role and a service account name that have been set up in advance.

Reference: https://docs.helm.sh/using_helm/#role-based-access-control
Steps:
Step 1 - if Tiller is installed, remove it
Step 2 - Create Tiller service account
Step 3 - Create a ClusterRoleBinding
Step 4 - Start Tiller with --service-account flag
