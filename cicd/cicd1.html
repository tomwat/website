MySkills website - the entire flow

High Level:
1. On my Developer workstation, I make a change to my website
2. I test the change locally on my workstation
3. I commit the change to my local git repo
4. I push the change to my remote repo - github.com
5. Jenkins is configured to listen for changes in my github repository
6. Jenkins initiates a build process - see below
7. At the end of the build process, a new website has been deployed to IBM Cloud

The build process:
The Jekinsfile will perform these 4 stages of build process:
(i) Build the Docker image when code is committed to GitHub.
(ii) Push the Docker image to the IBM Cloud Private Container registry.
(iii)Perform unit testing with container structure test framework.
(iv) Perform an upgrade of the Helm chart.
